## Known Limitations

### Multi-page OME-TIFF Files

Multi-page OME-TIFF files (microscopy data) crash during directory writing.
This appears to be a libtiff 4.5.1 limitation with complex metadata structures.

### Tiled Images with Specific Metadata

Some LZW-compressed tiled images with Page Number tags or XMP metadata crash.
Standard tiled images without complex metadata work correctly.

### Compression Level Tags

- DEFLATELEVEL tag causes crashes - currently disabled
- ZSTD level tag (65564) not supported in libtiff 4.5.1
- LZMA preset level works correctly

### Predictor

Horizontal predictor causes crashes with ZSTD/Deflate compression.
Currently disabled by default for stability.

## Recommendations

1. **For multi-page files:** Process pages individually if possible
2. **For tiled files:** Most work correctly; failures are metadata-specific
3. **For best compression:** Use ZSTD (default) with level 19
4. **For compatibility:** Use Deflate compression
